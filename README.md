# Análisis de Supervivencia del Titanic

## Descripción general del proyecto  
Este proyecto aborda el famoso caso del viaje del barco Titanic, que se hundió en 1912. Usando los datos cargados en la competencia de Kaggle, exploramos qué variables influyeron en la supervivencia de los pasajeros y construimos modelos de predicción. Nuestro objetivo es aprender a preparar datos, generar nuevas variables y entender patrones de supervivencia, además de aplicar técnicas básicas de machine learning.

## Integrantes y roles  
- William Gómez Marín – Data Wrangling y generación de variables nuevas  – Análisis exploratorio de datos y visualizaciones – Modelado predictivo, evaluación y entrega de resultados  

## Objetivos del trabajo  
1. Investigar cómo las características de los pasajeros (edad, sexo, clase, número de familiares a bordo, etc.) están relacionadas con su probabilidad de supervivencia.  
2. Crear nuevas variables que puedan aportar información adicional (por ejemplo: número de familiares, tamaño del grupo, etc.).  
3. Unificar los procesos de exploración, limpieza, modelado y evaluación en un solo flujo de trabajo reproducible.  
4. Generar conclusiones claras sobre qué factores pueden haber sido determinantes para la supervivencia.  

## Instrucciones básicas para ejecutar el código
Para revisar y reproducir nuestro trabajo, basta con seguir unos pasos sencillos.
Primero, se debe descargar o clonar el repositorio completo en el computador. Luego, es necesario asegurarse de que los archivos originales del concurso del Titanic estén ubicados en la carpeta de datos del proyecto. Una vez hecho esto, solo hay que abrir el archivo principal del análisis (notebook o script) y ejecutarlo desde el inicio hasta el final, siguiendo el orden en el que aparecen las celdas o bloques.
El proceso automáticamente carga los datos, crea las variables necesarias, realiza el análisis y muestra los resultados sin que el usuario tenga que modificar nada.

## Estructura del proyecto  
El repositorio está organizado de forma sencilla para facilitar su lectura.
En la carpeta de datos se almacenan los archivos originales del Titanic, necesarios para el análisis. Dentro de la carpeta de código se encuentra el notebook o los scripts que contienen todo el desarrollo del trabajo: la exploración de los datos, la creación de nuevas variables y el análisis final. También hay una carpeta destinada a los resultados generados, donde se guardan las tablas, gráficos o archivos producidos durante el proceso. Finalmente, el archivo README funciona como una guía general del proyecto y se muestra automáticamente al ingresar al repositorio.
